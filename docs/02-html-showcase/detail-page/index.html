Button

Buttons trigger actions or navigation. Use them for form submissions, dialogs, and primary user interactions.

Preview

Installation

npm install @dutchy/button

Usage

import { Button } from '@dutchy/button'

export default function Example() {
  return (
    <Button variant="primary">
      Click me
    </Button>
  )
}

Props

Prop Type Default Description
variant string "primary" Visual style variant
size string "md" Button size (sm, md, lg)
disabled boolean false Disable the button
loading boolean false Show loading spinner
onClick function - Click handler function

Variants

Primary

Use for primary actions and main CTAs.

Secondary

Use for secondary actions that complement primary buttons.

Destructive

Use for destructive actions like delete or remove.

FAQ

When should I use the Button component?
Use the Button component for any user-initiated action, such as form submissions, opening modals, navigating to new pages, or triggering side effects. Avoid using buttons for plain navigation â€” use links instead.
Can I customize the button styles?
Yes, the Button component accepts a className prop for custom styles. You can also extend the default variants through the design token configuration.
Does the Button support loading states?
Yes, pass the loading prop to display a spinner and disable the button. The button automatically prevents double-click submissions while loading.
